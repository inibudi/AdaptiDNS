# AdaptiDNS
Real-time adaptive bandwidth control system using hybrid rule-based and SVM-based web content classification via DNS log monitoring.

## About

AdaptiDNS is a real-time closed-loop adaptive bandwidth control system that integrates hybrid web content classification with automatic network policy enforcement. The system monitors DNS query logs generated by dnsmasq, classifies accessed domains using a two-layer hybrid approach (rule-based fast-path and SVM-based slow-path), and automatically applies graduated bandwidth penalties to users who access negative content (gambling, pornography, phishing) via the MikroTik RouterOS API.

This system was developed as part of doctoral dissertation research at Universitas Universitas Andalas, Padang, Indonesia.

## Key Features

- Real-time DNS log parsing with asynchronous tail-based monitoring
- Hybrid classification: rule-based fast-path + SVM slow-path (90.50% accuracy)
- Graduated penalty enforcement via MikroTik RouterOS API
  - Pornography  : bandwidth reduced to 5M/5M  (50% reduction)
  - Gambling     : bandwidth reduced to 7M/7M  (30% reduction)
  - Phishing     : bandwidth reduced to 9M/9M  (10% reduction)
- Automatic bandwidth restoration after 10-minute penalty duration
- Real-time monitoring dashboard (Flask + Socket.IO)
- IP-to-username mapping via MikroTik Hotspot API
- MySQL-based logging for audit and analysis

## System Requirements

- Ubuntu Server 20.04 LTS
- Python 3.8+
- MikroTik RouterOS 6.48+ (tested on RB941-2nD)
- MySQL 8.0+
- dnsmasq 2.8+
